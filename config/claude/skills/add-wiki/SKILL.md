# add-wiki

현재 repo의 `docs/` 아래에 일관된 문서 스키마를 강제하면서 (1) 신규 문서 생성 또는 (2) 기존 문서 보강을 수행한다.

## 0. 사전 확인

- 현재 작업 디렉토리(혹은 사용자가 지정한 repo 루트)에서 `docs/` 폴더 존재 여부를 확인한다.
- `docs/` 가 없으면 사용자에게 경로를 한 번 묻고, 그래도 없으면 생성 여부를 확인한다.
- 프로젝트 루트에 `CLAUDE.md` 가 있으면 한 번 읽고 그 규칙을 따른다. 본 SKILL.md 와 충돌하면 `CLAUDE.md` 가 우선이다.

## 1. 입력 분류

사용자 입력을 보고 둘 중 하나로 분류한다.

| 종류 | 신호 | 동작 |
|---|---|---|
| **신규** | raw 텍스트만 있고 어디 들어갈지 미정 | 새 파일 생성 |
| **보강** | 사용자가 특정 .md 경로를 지정 / 같은 도메인 문서가 이미 존재 | 기존 파일에 ## 섹션 추가 또는 본문 갱신 |

판단이 애매하면 사용자에게 한 번만 묻는다.

## 2. 카테고리 결정

`docs/` 하위 폴더 구조는 repo마다 다르다. 다음 순서로 결정한다.

1. `docs/` 하위에 이미 존재하는 폴더 목록을 읽는다.
2. 입력 내용이 기존 폴더 중 어디에 맞는지 판단한다.
3. 기존 폴더 어디에도 명확히 안 맞으면 새 폴더를 제안한다.

**새 폴더 제안 조건**:
- "왜 기존 폴더 어디에도 안 맞는지" 한 줄로 설명한다.
- 새 카테고리에 최소 2~3개 문서가 모일 것 같은지 가늠한다. 한 문서만을 위한 폴더는 만들지 않는다.

**폴더가 아예 없는 경우**: 내용 성격에 따라 첫 폴더 구조를 제안하고 사용자 확인 후 생성한다.

## 3. 스키마 강제 (필수)

모든 신규/수정 .md 는 다음 규칙을 만족해야 한다.

### 3.1 frontmatter

```
---
title: 문서 제목
description: 한 줄 요약 (어떤 내용인지, 누가 보는 문서인지)
---
```

- `title`, `description` 둘 다 필수.
- 따옴표 없이 plain string.

### 3.2 본문 구조

- `##` 섹션 단위로 끊는다. 한 섹션이 너무 길면 둘로 나눈다.
- `###` 는 ## 안에서만 쓴다.
- 코드/명령어/경로는 백틱 또는 코드 블록으로.
- 표는 간결하게, 필요할 때만.

### 3.3 사실만

- "일반적으로~", "보통~", "대부분~" 같은 추측·일반론 금지.
- raw 입력에 그런 표현이 섞여 있으면 → 사실로 검증 가능한 부분만 남긴다.
- 출처가 불확실하면 사용자에게 한 번 확인하고, 끝까지 불확실하면 그 줄은 뺀다.

### 3.4 이미지

- `docs/images/` 에 두고 상대 경로로 참조: `![설명](../images/foo.png)`.

## 4. 신규 vs 보강

### 신규

- 파일 경로: `docs/<category>/<slug>.md`
- `<slug>` 은 영문 케밥-케이스 (예: `git-flow`, `nats-subject`).
- 이미 같은 slug 이 있으면 사용자에게 "보강이 맞는지" 확인.

### 보강

- 기존 파일의 적절한 ## 섹션을 찾아 본문 추가/수정.
- 없는 주제면 새 ## 섹션을 끝에 추가.
- frontmatter 의 `title` 은 보존, `description` 은 정확도가 더 높아지면 갱신 가능.
- 원본 파일의 톤·도메인 용어를 따른다.

## 5. 저장 후 보고

저장 직후 한 줄로 요약한다.

> `docs/domain/starfruit-back.md` 신규 생성: ## 역할, ## 처리 이벤트, ## 통신 구조

## 안티패턴 (절대 안 함)

- frontmatter 없이 그냥 `# 제목` 으로 시작하는 .md 만들기
- "일반적으로", "보통", "대부분" 같은 추측 문장 그대로 옮기기
- 한 ## 섹션에 너무 많은 내용 욱여넣기 (한 섹션이 화면 두 개 넘기면 분리)
- `docs/` 외부에 위키 문서 두기
- 카테고리 안 맞는데 가까운 폴더에 억지로 끼워 넣기
- raw 입력의 Slack 멘션·이모지·타임스탬프를 그대로 남기기
