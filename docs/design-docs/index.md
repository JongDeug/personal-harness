# Design Docs Index

설계 문서 목록. 새로운 시스템이나 주요 변경 사항에 대해 설계 문서를 작성한다.

## 작성 가이드

설계 문서는 다음 구조를 따른다:

1. **제목 & 날짜**
2. **배경** — 왜 이 변경이 필요한가
3. **목표 / 비목표** — 이번에 해결할 것과 하지 않을 것
4. **설계안** — 구체적인 기술 설계
5. **대안 검토** — 고려했지만 선택하지 않은 방안과 이유
6. **리스크** — 알려진 위험 요소
7. **마일스톤** — 단계별 실행 계획
