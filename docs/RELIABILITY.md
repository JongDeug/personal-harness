# Reliability Guide

서비스 안정성과 모니터링 가이드라인.

## 안정성 원칙

1. **장애는 일어난다** — 예방보다 빠른 감지와 복구에 집중
2. **관찰 가능성** — 로그, 메트릭, 트레이싱으로 시스템 상태를 파악 가능해야 한다
3. **점진적 배포** — 한 번에 전체 배포하지 않는다

## 로깅

### 규칙

- 구조화된 로그 (JSON 형식)
- 로그 레벨: `error` > `warn` > `info` > `debug`
- 프로덕션에서는 `info` 이상만 출력
- 민감 정보(비밀번호, 토큰 등) 로그에 포함 금지

### 필수 로그 포인트

- API 요청/응답 (status, latency)
- 인증 실패
- 외부 서비스 호출 실패
- 예상치 못한 에러

## 모니터링

### 필수 메트릭

- 서버 응답 시간 (p50, p95, p99)
- 에러율 (5xx 비율)
- 서비스 가동률
- DB 쿼리 성능

### 알림

- **즉시**: 5xx 에러율 > 5%, 서비스 다운
- **주의**: 응답 시간 p95 > 1s, 디스크/메모리 > 80%

## 장애 대응

1. **감지**: 알림 또는 사용자 리포트
2. **완화**: 임시 조치 (롤백, 기능 비활성화)
3. **수정**: 근본 원인 해결
4. **회고**: 무엇이 잘못되었고, 어떻게 방지할 것인지 기록

## 백업

- 중요 설정: Git으로 버전 관리
- 백업 복구 테스트: 분기별 1회
