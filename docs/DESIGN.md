# Design Guide

UI/UX 디자인에 대한 조직 수준 가이드라인.

## 디자인 원칙

1. **일관성** — 같은 패턴은 같은 방식으로 표현한다
2. **명확성** — 사용자가 다음 행동을 즉시 알 수 있어야 한다
3. **최소주의** — 불필요한 요소를 제거한다. 모든 요소에는 역할이 있어야 한다
4. **접근성** — 기본적인 접근성 기준을 충족한다 (명암비, 터치 타겟 등)

## 디자인 토큰

프로젝트별로 디자인 토큰을 정의하되, 조직 공통 규칙:

### 간격 (Spacing)

4px 단위 시스템: `4, 8, 12, 16, 20, 24, 32, 40, 48, 64`

### 반응형 브레이크포인트

| 이름 | 크기 | 대상 |
|------|------|------|
| `sm` | 640px | 모바일 |
| `md` | 768px | 태블릿 |
| `lg` | 1024px | 데스크톱 |
| `xl` | 1280px | 와이드 |

### 다크 모드

- 모든 프로젝트에서 다크 모드를 지원한다
- 시스템 설정을 기본으로 따르되, 수동 전환 가능

## 컴포넌트 설계 규칙

- Atomic Design 기반: Atoms → Molecules → Organisms → Templates → Pages
- 컴포넌트는 독립적으로 동작해야 한다 (외부 상태 의존 최소화)
- 프로젝트별 디자인 시스템 레퍼런스: `references/design-system-reference-llms.txt`
