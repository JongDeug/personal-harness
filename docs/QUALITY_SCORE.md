# Quality Score

코드 및 제품 품질을 측정하고 유지하기 위한 기준.

## 코드 품질 체크리스트

PR 머지 전 self-review:

### 필수

- [ ] 의도한 대로 동작하는가 (직접 테스트)
- [ ] 타입 에러, lint 에러 없음
- [ ] 보안 취약점 없음 (입력 검증, SQL injection, XSS 등)
- [ ] 기존 테스트 통과
- [ ] 커밋 메시지가 Conventional Commits 형식

### 권장

- [ ] 핵심 로직에 대한 테스트 추가
- [ ] 불필요한 console.log / print 제거
- [ ] 하드코딩된 값 없음 (환경 변수 또는 상수로)
- [ ] 에러 핸들링이 적절한가
- [ ] 디자인 패턴이 forces에서 자연스럽게 도출되었는가 (억지 적용 아닌지, 단순 코드에 과도한 패턴이 씌워지지 않았는지)

## 프로젝트 건강도 지표

정기적으로 확인할 항목:

| 지표 | 목표 | 확인 주기 |
|------|------|-----------|
| CI 성공률 | > 95% | 주간 |
| 테스트 커버리지 | > 60% (핵심 경로) | 월간 |
| 빌드 시간 | < 5분 | 월간 |
| 의존성 취약점 | 0 critical | 주간 |
| 번들 사이즈 증가율 | < 5%/월 | 월간 |
